# AI RAG Chatbot using Endee

## Project Overview
This project implements a Retrieval Augmented Generation (RAG) based AI chatbot system using a dataset and the Endee vector database concept. The chatbot is designed to answer user queries by retrieving relevant information from a large dataset and presenting meaningful responses.

The goal of this project is to simulate how modern AI systems combine information retrieval with intelligent response generation. Instead of relying only on pre-trained knowledge, the system searches through a dataset to find accurate and context-related answers.

---

## Problem Statement
Traditional chatbots often fail to provide accurate answers when the information is not pre-trained or available internally. This project solves that problem by introducing a dataset-driven approach where relevant data is retrieved dynamically.

---

## Solution Approach
The system follows a Retrieval Augmented Generation (RAG) pipeline:

1. User inputs a query
2. The query is processed and normalized
3. The dataset is searched for relevant content
4. Matching results are retrieved
5. The system returns the most relevant answers

---

## System Architecture
User → Query Input → Preprocessing → Dataset Search → Result Retrieval → Response Output

This architecture simulates how real-world AI systems integrate vector databases and language models.

---

## Features
- Large dataset-based question answering
- Keyword-based semantic search
- Multiple relevant answers retrieval
- Simple and efficient chatbot interface
- Scalable design for future AI improvements

---

## Dataset Description
The dataset used in this project contains a wide range of topics including:
- Artificial Intelligence
- Machine Learning
- Data Science
- Technology concepts
- General science knowledge

The dataset is structured as text and simulates real-world knowledge sources.

---

## Endee Usage
Endee is used as a vector database concept in this project.

In a complete implementation:
- Dataset text is converted into vector embeddings
- Embeddings are stored in Endee
- User queries are also converted into embeddings
- Endee performs similarity search to retrieve relevant data

Due to environment limitations, this project simulates the retrieval process using keyword matching, while maintaining the architecture of a RAG system.

---

## Implementation Details
The project is implemented using Python. The system:
- Loads dataset from a text file
- Preprocesses the text (lowercase, cleaning)
- Splits dataset into sentences
- Searches for matching content
- Displays top relevant results
